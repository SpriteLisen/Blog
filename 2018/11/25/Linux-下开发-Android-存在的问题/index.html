Linux 下开发 Android 存在的问题

Posted on 2018-11-25 Edited on 2018-12-04 In Android , Linux , 未知问题

近期在 Linux 下开发 Android, 发现一个奇怪的问题, 目前不知产生原因, 特此记录, 多加注意.

  1. 前因

  这段时间刚搭建好了 Elementary OS 系统 (基于 Ubuntu), 想着用 Linux 做主开发环境, 因为在 Windows 上开发用着 AS 实在是有点太卡了, 特别是编译的时候, 简直是在燃烧生命.

  刚开始用的时候没特别在意, 一顿操作就把 Windows 下的项目移植到了 Linux 下, 并搭建了 Gitlab 服务器, 把代码都提交到了服务器上. 兴高采烈的开发着.

  1. 过程

  就这样在 Linux 继续着我的开发大业, 并搭建了一下本地的基类代码仓库, 想着以后都不要再从基本开始搭建了, 然后在现有的项目中一阵狂改, 跑起来之后 发现 RecyclerView 滑动起来有些卡顿, 特别是在 item 数量比较多的情况下 (超过一屏), 也不是特别卡, 因为开发中的简记是个本地项目, 无需联网, 想着这点卡顿也影响不到用户体验, 也就这么算了.

  后来开始把 智享校园 的项目也移到的 Linux 上来继续开发, 并开始使用 Kotlin 进行整个项目的推进, 从中学习 Kotlin. 这一试不得了, 一开始拉取课表, 整个界面都卡住了, 按理说 Retrofit 是在 IO 线程中进行数据拉取的, 不会影响到主线程才对, 这 接近 ANR 的现象 让我着实懵逼.

  不明所以的我以为是 Kotlin 的执行效率比 Java 低, 因为感觉毕竟是 JVM 语言, 最终还是要转换为 Java 字节码, 还在 Java 的基础上添加了一堆判空的代码, 所以我断然认定 Kotlin 的执行效率是要比 Java 低的, 还在心中暗骂: 这种垃圾语言怎么这么多人吹???

  不过我心中一直有个疑问, 就是为何在 IO 线程中的数据请求, 竟然影响到了主线程.

  直到最近, 我决定放弃继续使用 Kotlin 开发 Android, 继续使用 Java, 并将原先 Java 开发的项目移动到 Linux 上来编译. 这时候才发现了事情的真相. 我发现同样的代码, 同样的逻辑, Kotlin 和 Java 编写的版本执行效率都非常低下. 这让我觉得更加蹊跷了, 早之前好像还没这种问题, 最近也就更新了下 AS, 从 3.1.1 更新到了 3.2.1, 我就想着是不是由于新版本 AS 的原因导致了这个问题.

  伴随着疑问的我, 在百度上搜索着却毫无答案, 无助的我选择了在大神群中提问, 看各位大佬有没有遇到过这种更新 3.2.1 导致执行效率变低的问题. 得到的结论是 :

“AS 只是一个工具, 他不具备更改代码的行为. 实在不行你可以退回到 3.0 版本试一下”.

  带着这个结果, 于是我在 Windows 下把代码编译了一遍. (因为此时我的 Windows AS 的版本是 3.0.1), 之后的结果是不卡, 没有这种问题.

  于是我在 Linux 下也下载了 AS 3.1.1 版本, 编译之后发现还是很卡, 我又继续下载了 3.0.1 版本, 编译之后还是很卡. 这时我意识到并不是 AS 版本的问题了, 好像是关系到系统的问题. 我在 Linux 上 发布了一个 release 版测试了一下, 发现并不会卡顿, 但是一旦 用 debug 版本的时候, 就卡的不行.

  我又将 Android SDK 也重新下载了一遍, 发现也是一样的, 这也不关 SDK 版本的问题.

  带着这个现象, 我又到百度上去搜索了一番, 得到的结论是这样 :

“debug 版本往往携带着很多断点策略代码, 并且没有优化, 而 release 版本是删去了这些代码并且进行过优化的, 自然 debug 版本的效率没有 release 版本的高”

  虽然得到了这样的答案, 但是想着还是觉得有点不对. 如果是这种说法, 那 为什么在 Windows 上 debug 版本的效率却又不会受到丝毫影响呢?

  1. 结论

  在上述的基础上, 苦寻无果.
  于是我做了一个推导 :

假设 debug 版本是携带了很多调试代码的, 那么出现这种系统区别导致运行效率降低的情况可能只有一种, 那就是 Linux 下的调试代码比 Windows 下要复杂且执行步骤更多, 因此效率上也会比 Windows 低得多.

  这个假设没有得到证实, 这仅仅只是我的猜测, 目前没有更多的时间去证实这个事情, 好像其他使用 Linux 开发 Android 的开发者也没说过出现过类似的情况, 我就猜想也许还存在 Linux 版本的问题, 之后有时间测试下在别的 Linux 版本上是否也存在该问题.
