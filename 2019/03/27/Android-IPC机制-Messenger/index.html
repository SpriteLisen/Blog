Android IPC机制-Messenger

About 7 minutes for read Posted on 2019-03-27 Edited on 2019-04-29 In Android , IPC

Android 中的 IPC 机制有很多, 如: Intent 附加 Bundle 数据, Binder, 通过 AIDL 快速实现 Binder, Messenger, ContentProvider, Socket, 共享文件 等等

今天主要聊一聊 Messenger 对象, 一个轻量级的 IPC 实现方案.

  1. 剖析 Messenger 对象

  Messenger 译为 送信者, 通过它, 我们可以轻松的在不同进程间传递数据.

  我们原先在子线程中向 UI 线程发送更新 UI 等消息, 都是通过 Handler.sendMessage() 来发送一个 Message 对象来实现的.

  我们知道在 Message 对象中, 我们可以 通过 what 属性来设置消息类别, obj 属性来携带需要传递的数据, 除此之外我们还可以 通过 setData(Bundle data) 方法来传递带有数据的 Bundle 对象, 借此来传递更多类型的数据给 UI 线程.

  此次, 我们在使用 Messenger 对象来实现 IPC 时, 也是借助 Message 对象来传递我们的数据.

  Messenger 是一种轻量级的 IPC 实现方案, 我们可以通过该类的源码发现, 它的底层实现就是 AIDL, 通过 AIDL 来实现 Binder 来达到跨进程通信的目的.

  1. 使用 Messenger 对象

  Messenger 的使用方式与 Socket 类似, 分为服务器与客户端两方, 与 Socket 不同的是它一次只处理一个请求, 所以它不存在并发执行的问题, 我们也就无需像共享文件一样去处理线程同步的问题了.

服务器端

  一般我们用 Service 来充当服务器, 我们首先构造一个 Messenger 对象, 在构造方法中传入我们处理消息的自定义 Handler 对象即可, 接下来我们只需要在 Service 的 onBind 方法中将 Messenger 持有的 Binder 对象返回给 客户端 即可.

客户端

  我们在客户端中绑定服务器端的 Service, 然后利用绑定成功时返回的 IBinder 对象来创建一个 Messenger 对象, 这样我们就可以通过该 Messenger 对象来传递数据给服务器端的 Service 了.

  1. 双向通讯案例

  与单向不同, 单向的案例是只有服务器能够接收并处理消息, 客户端若是也想接收并处理消息, 就也需要自定义处理消息的 Handler 类, 然后与服务器一样, 借助该 Handler 对象来构建一个 Messenger 对象, 再借助 Message.replyTo 属性, 将自己的 Messenger 对象传递给服务器, 服务器就可以通过该 Messenger 对象来向客户端发送消息了.
